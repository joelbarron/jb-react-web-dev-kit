# Release Guide (npm + Changesets)

This project uses Changesets with two channels:
- `latest` from branch `main` (stable)
- `next` from branch `next` (rc prerelease)

## One-time setup

1. Configure npm Trusted Publisher for package `@joelbarron/react-web-dev-kit`.
2. Point it to this GitHub repository and workflow file `.github/workflows/release.yml`.
3. Remove `NPM_TOKEN` from repository secrets after Trusted Publisher is validated.

Why: publish now uses OIDC (`id-token: write`) and no npm token.

## Branch bootstrap for prerelease channel

Run once to initialize `next` in prerelease mode:

```bash
git checkout main
git pull
git checkout -b next
npx changeset pre enter rc
git add .changeset/pre.json
git commit -m "chore: enter rc prerelease mode on next"
git push -u origin next
```

Do not commit `.changeset/pre.json` into `main`.

## Daily release flow

1. Work on `develop` and commit your code changes.
2. Choose release mode:
   - Stable: run `npm run release:stable` (merges `develop -> main` and pushes).
   - RC: run `npm run release:rc` (merges `develop -> main -> next` and pushes).
3. If no pending changeset exists, the script runs `npx changeset add` and commits it automatically.
4. Wait for GitHub Actions to update/create the release PR:
   - `changeset-release/main` for stable.
   - `changeset-release/next` for rc.
5. Merge the release PR to publish.

## Helper scripts

You can use these commands instead of running git commands manually:

```bash
# Stable flow from develop:
# - if no pending changeset exists, runs `npx changeset add` and commits it
# - pushes develop
# - merges develop into main and pushes
npm run release:stable

# RC flow from develop:
# - if no pending changeset exists, runs `npx changeset add` and commits it
# - merges develop into main
# - merges main into next
# - pushes main and next
npm run release:rc
```

## When version changes in package.json

You do not edit `package.json` version manually for releases.

- The version bump is generated by Changesets when the workflow runs `changeset version`.
- That change appears in the `changeset-release/main` or `changeset-release/next` PR.
- When you merge that PR, the new `package.json` version is committed and published.

## Validation

Check tags and versions:

```bash
npm view @joelbarron/react-web-dev-kit version dist-tags --json
```

Expected:
- `latest` remains stable
- `next` points to the newest rc prerelease
